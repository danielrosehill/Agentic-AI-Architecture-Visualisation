**Date**: 2026-03-10

## Stats
- Layers: 14 (up from 13)

## What Changed

### Human Guidance (Daniel's feedback on v2.0 visualization)

Three observations drove this iteration:

1. **"An LLM gateway is optional for using LLMs."** The v2.0 diagram implied all inference routes through the gateway. In reality, many deployments call cloud APIs or self-hosted endpoints directly. We now show dual paths: "VIA GATEWAY" and "DIRECT".

2. **"Grounding should be its own bubble, linking off to search APIs."** Search APIs were buried inside the Actions layer alongside CRM data and payment systems. Grounding (web search, news feeds, real-time data) is architecturally distinct — it's an *input* that clusters around the agent, not an *action* the agent takes on the world. Separated into its own layer.

3. **"Storage should be off to the side."** Having Storage at the bottom of the center column implied it was the final step in a sequential chain. It's not — it's a persistence layer that agents write to asynchronously. Moved to a side position like Frontends and Safety.

### New Layer
- `grounding` — Real-time external data sources (web search, news feeds, live APIs) that ground agent responses in current information. Positioned as an input to agents.

### Removed from Actions
- `search-apis` — Moved conceptually into the Grounding layer

### New Connections
- `direct-to-cloud-apis` — Direct inference path bypassing the gateway
- `direct-to-self-hosted` — Direct inference path bypassing the gateway
- `grounding-to-agents` — Grounding data feeds into agent reasoning

### Layout Changes
- Storage moved from center column (bottom) to side position
- Inference layer now shows optional gateway with dual routing paths
- Grounding rendered as a side card near the Agents layer

## Design Principle
These changes reflect a shift from "sequential pipeline" to "hub-and-spoke around agents" — the agent is the center, with inputs (grounding, context, inference) clustering around it and outputs (storage, actions, frontends) radiating outward.
